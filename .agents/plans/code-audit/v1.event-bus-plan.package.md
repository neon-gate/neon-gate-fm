## Findings

| Rule | Status | Location |
|------|--------|----------|
| Semantic Suffixes | FAIL | `event-handler.ts` exports a type; should be `event-handler.type.ts` |
| Semantic Suffixes | MINOR | `errors.ts` - no suffix; consider `event-bus.errors` or similar |
| Semantic Suffixes | MINOR | `create-event-bus.ts` - factory function; no standard suffix in table |
| Prefer Absolute Imports | FAIL | `nats-queue-consumer.adapter.ts:3` - `from '../event-handler'` uses `../` relative import |
| Ports As Abstract Classes | FAIL | `nats.adapter.ts:5`, `noop-event-bus.adapter.ts:5` - `implements EventBus<Events>` but `EventBus` is an abstract class; should use `extends` |

## Remediation

1. Rename `event-handler.ts` to `event-handler.type.ts` and update all imports.
2. (Optional) Rename `errors.ts` to `event-bus-error.ts` (single error class) or add `.types` if adding more error types.
3. (Optional) Document `create-event-bus` as acceptable factory pattern for package entry point.
4. Change `nats-queue-consumer.adapter.ts:3` relative import to use an absolute path or same-level `./` import.
5. Change `NatsEventBusAdapter` and `NoopEventBusAdapter` from `implements EventBus<Events>` to `extends EventBus<Events>`.
