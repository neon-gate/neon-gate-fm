# `@pack/nats-broker-messaging` v1 Refactoring Plan

## Invariant

**`@pack/kernel` is never modified.** It is the single source of truth for domain abstractions.

---

## 1. Problem Statement

The current `@pack/nats-broker-messaging` package duplicates kernel abstractions and has a flat, inconsistent architecture:

- `event-bus.abstract.ts` defines its own `EventBus` class (emit/on) that shadows kernel's `EventBus` (subscribe/publish)
- `event-handler.ts` defines a function-type handler; kernel already has an abstract class `EventHandler`
- `event-map.ts` defines an interface; kernel already has an abstract class `EventMap`
- `errors.ts` extends plain `Error` instead of kernel's `DomainError`
- NATS adapters use `DomainEventPrimitive` which doesn't exist in kernel; kernel exports `EventPrimitive`
- No separation between the **publisher** concern (emit to NATS) and the **consumer** concern (subscribe from NATS with queue groups)
- Adapter files live in inconsistent folders (`adapters/` vs `nats/`)

---

## 2. Kernel Abstractions (read-only reference)

These are the kernel types that event-bus must conform to:

### EventPrimitive (serialized envelope)

```ts
interface EventPrimitive<Payload = Record<string, unknown>> {
  eventId: string
  eventName: string
  eventVersion: number
  aggregateId: string
  occurredOn: Date
  payload: Payload
}
```

### DomainError

```ts
class DomainError extends Error {
  readonly code: string
  constructor(code: string, message: string)
}
```

### EventMap (abstract class)

```ts
abstract class EventMap<EventDomain> {
  abstract mapEvents(events: ObjectPrimitive<EventDomain>[]): ObjectPrimitive<EventDomain>[]
}
```

> Note: Kernel's `EventMap` is an abstract class with `mapEvents()`. Domain services currently use `interface XEventMap extends EventMap { [key: string]: payload }` as an index-signature map. This interface pattern is the real contract consumed by NATS adapters. Since we cannot modify kernel, we define a local `EventContract` type to serve as the adapter constraint while keeping `EventMap` from kernel as the re-export for backward compatibility.

### 3.3 Define EventContract type

Create `src/types/event-contract.type.ts`:

```ts
/**
 * A simple event-name-to-payload mapping used by NATS adapters.
 * Domain services define their contracts as interfaces extending this type.
 *
 * @example
 * interface AuthorityEventContract extends EventContract {
 *   [AuthorityEvent.UserSignedUp]: { userId: string; email: string }
 * }

export interface EventContract {
  [eventName: string]: Record<string, unknown>
}
```

This is the adapter-level constraint. It replaces the deleted `event-map.ts` interface without conflicting with kernel's abstract `EventMap` class.

- Replace all `DomainEventPrimitive` with `EventPrimitive` from `@pack/kernel`
- Replace all local `EventMap` imports with `EventContract` for adapter usage
- Re-export kernel's `EventMap` from the package index for backward compatibility

### 4.2 NatsPublisher — emit events to NATS

Replaces the current `NatsEventBusAdapter` emit concern. Accepts `EventPrimitive` as the transport envelope.

```ts
import { type NatsConnection, StringCodec } from 'nats'
import type { EventPrimitive } from '@pack/kernel'
import type { EventContract } from '../types/event-contract.type'
import { EventBusPublishError } from '../event-bus.error'

export class NatsPublisher<Events extends EventContract> {
  private readonly sc = StringCodec()

  constructor(private readonly nc: NatsConnection) {}

  async publish<E extends keyof Events & string>(
    subject: E,
    envelope: EventPrimitive<Events[E]>
  ): Promise<void> {
    try {
      const data = this.sc.encode(JSON.stringify(envelope))
      this.nc.publish(subject, data)
    } catch (err) {
      throw new EventBusPublishError(subject, String(err))
    }
  }
}
```

**Key design decisions:**

- The method is named `publish` (not `emit`) to align with NATS vocabulary
- It takes an `EventPrimitive` envelope, meaning the caller serializes the domain event via `event.toPrimitive()` before publishing — this keeps the adapter unaware of domain event classes
- The generic constraint `Events extends EventContract` preserves type-safe subject-to-payload mapping

### 4.3 NatsConsumer — subscribe to NATS subjects with queue groups

Replaces both `NatsEventBusAdapter.on()` and `NatsQueueConsumerAdapter`. A single class that always uses queue groups (the standard for competing consumers).

```ts
import { type NatsConnection, StringCodec } from 'nats'
import type { EventPrimitive } from '@pack/kernel'
import type { EventContract } from '../types/event-contract.type'
import { EventBusSubscriptionError } from '../event-bus.error'

export type MessageHandler<Payload> = (
  envelope: EventPrimitive<Payload>
) => void | Promise<void>

export class NatsConsumer<Events extends EventContract> {
  private readonly sc = StringCodec()
  private readonly unsubscribers: Array<() => void> = []

  constructor(
    private readonly nc: NatsConnection,
    private readonly queue: string
  ) {}

  subscribe<E extends keyof Events & string>(
    subject: E,
    handler: MessageHandler<Events[E]>
  ): () => void {
    const sub = this.nc.subscribe(subject, { queue: this.queue })

    void (async () => {
      for await (const msg of sub) {
        try {
          const decoded = JSON.parse(
            this.sc.decode(msg.data)
          ) as EventPrimitive<Events[E]>
          await handler(decoded)
        } catch {
          // Per-message errors must not break the subscription loop.
          // Caller's handler is responsible for error reporting.
        }
      }
    })()

    const unsubscribe = () => sub.unsubscribe()
    this.unsubscribers.push(unsubscribe)
    return unsubscribe
  }

  unsubscribeAll(): void {
    for (const fn of this.unsubscribers) fn()
    this.unsubscribers.length = 0
  }
}
```

**Key design decisions:**

- Always uses queue groups — this is the standard pattern for Pulse services
- Handler receives `EventPrimitive<Payload>` (the full envelope), not just the raw payload. This gives consumers access to `eventId`, `aggregateId`, `occurredOn`, etc.
- No dependency on kernel `EventHandler` class — consumers are NestJS services that call use cases, not DDD event handlers

### 4.4 NoopPublisher and NoopConsumer

Simple no-ops for local development when NATS is not available.

```ts
export class NoopPublisher<Events extends EventContract> {
  async publish<E extends keyof Events & string>(
    _subject: E,
    _envelope: EventPrimitive<Events[E]>
  ): Promise<void> {}
}

export class NoopConsumer<Events extends EventContract> {
  subscribe<E extends keyof Events & string>(
    _subject: E,
    _handler: MessageHandler<Events[E]>
  ): () => void {
    return () => undefined
  }

  unsubscribeAll(): void {}
}
```

### 4.5 NestJS infrastructure (unchanged)

These files stay as-is with minimal import updates:

- `nats-connection.provider.ts` — provides `NatsConnection | null` via DI
- `nats-lifecycle.service.ts` — drains connection on module destroy
- `nats-config.enum.ts` — renamed from `event-bus-config-flag.enum.ts` for clarity

### 4.6 createEventBus factory (removed)

The current `createEventBus()` function creates an in-memory event bus. This concern belongs to kernel's `EventBus` (in-process domain dispatch), not to the inter-service messaging package. It is deleted.

If services need an in-memory bus for testing, they should use kernel's `EventBus` directly.

## 6. Migration Impact on Domain Services

```ts
// BEFORE
import { EventBus } from '@pack/nats-broker-messaging'
export abstract class AuthorityEventBusPort extends EventBus<AuthorityEventMap> {}

// AFTER
import { NatsPublisher } from '@pack/nats-broker-messaging'
export abstract class AuthorityEventBusPort extends NatsPublisher<AuthorityEventMap> {}
```

### Provider factories (before -> after)

```ts
// BEFORE
return new NatsEventBusAdapter<AuthorityEventMap>(connection)

// AFTER
return new NatsPublisher<AuthorityEventMap>(connection)
```

### Use case publish (before -> after)

```ts
// BEFORE
void this.events.emit(AuthorityEvent.UserLoggedIn, loginEvent.toPrimitive())

// AFTER
void this.events.publish(AuthorityEvent.UserLoggedIn, loginEvent.toPrimitive())
```

### Consumer wiring (before -> after)

```ts
// BEFORE
this.consumer = new NatsQueueConsumerAdapter<TrackUploadedEventMap>(connection, queue)

// AFTER
this.consumer = new NatsConsumer<TrackUploadedEventMap>(connection, queue)

  // envelope.payload contains the typed payload
  // envelope.eventId, envelope.aggregateId, etc. are available
  await this.generateFingerprint.execute({ ... })
})
```

## 9. Phase V1.1 — Envelope Validation and Structured Errors

**Goal:** Runtime safety for decoded envelopes; stop trusting `as` casts.

### 9.1 New error types

Add to `event-bus.error.ts`:

```ts
export class EventBusValidationError extends DomainError {
  constructor(
    subject: string,
    readonly field: string,
    readonly reason: string
  ) {
    super(
      'EVENT_BUS_VALIDATION_ERROR',
      `Invalid envelope on "${subject}": ${field} - ${reason}`
    )
    this.name = 'EventBusValidationError'
  }
}

export class EventBusVersionMismatchError extends DomainError {
  constructor(
    subject: string,
    readonly expected: number,
    readonly received: number
  ) {
    super(
      'EVENT_BUS_VERSION_MISMATCH',
      `Event "${subject}" version mismatch: expected ${expected}, received ${received}`
    )
    this.name = 'EventBusVersionMismatchError'
  }
}
```

### 9.2 Envelope schema validation

Create `src/validation/envelope-schema.ts` using Zod (or similar):

```ts
import { z } from 'zod'

export const eventPrimitiveEnvelopeSchema = z.object({
  eventId: z.string().uuid(),
  eventName: z.string().min(1),
  eventVersion: z.number().int().positive(),
  aggregateId: z.string().min(1),
  occurredOn: z.union([z.string().datetime(), z.date()]).transform((v) => (typeof v === 'string' ? new Date(v) : v)),
  payload: z.record(z.unknown())
})

export type ValidatedEnvelope = z.infer<typeof eventPrimitiveEnvelopeSchema>
```

### 9.3 Validation in NatsConsumer

Before invoking the handler, validate the decoded envelope:

```ts
const decoded = JSON.parse(this.sc.decode(msg.data)) as unknown
const result = eventPrimitiveEnvelopeSchema.safeParse(decoded)
if (!result.success) {
  // Log or emit to dead-letter; do not throw (loop must not break)
  this.onValidationError?.(subject, result.error)
  return
}
await handler(result.data as EventPrimitive<Events[E]>)
```

### 9.4 Optional version check

Allow consumers to declare expected `eventVersion` per subject:

```ts
subscribe<E extends keyof Events & string>(
  subject: E,
  handler: MessageHandler<Events[E]>,
  options?: { expectedVersion?: number }
): () => void
```

If `expectedVersion` is set and `envelope.eventVersion !== expectedVersion`, reject the message (log or dead-letter) instead of invoking the handler.

### 9.5 Dependency

Add `zod` as a dependency (or keep validation optional if Zod is not desired in the package).

## 10. Phase V1.2 — Middleware Pipeline

**Goal:** Composable cross-cutting concerns (logging, tracing, idempotency) without touching adapters.

### 10.1 Middleware types

Create `src/middleware/types.ts`:

```ts
import type { EventPrimitive } from '@pack/kernel'

export type PublishContext = {
  subject: string
  envelope: EventPrimitive
  timestamp: number
}

export type ConsumeContext = {
  subject: string
  envelope: EventPrimitive
  timestamp: number
}

export type PublishMiddleware = (
  ctx: PublishContext,
  next: () => Promise<void>
) => Promise<void>

export type ConsumeMiddleware = (
  ctx: ConsumeContext,
  next: () => Promise<void>
) => Promise<void>
```

### 10.2 Middleware composition

```ts
export function composePublishMiddleware(
  ...middlewares: PublishMiddleware[]
): (ctx: PublishContext, next: () => Promise<void>) => Promise<void> {
  return async (ctx, next) => {
    let index = 0
    const run = async (): Promise<void> => {
      if (index >= middlewares.length) return next()
      const mw = middlewares[index++]
      return mw(ctx, run)
    }
    return run()
  }
}
```

Same pattern for `consumeMiddleware`.

### 10.3 Built-in middleware

| Middleware | Purpose |
|------------|---------|
| `LoggingPublishMiddleware` | Log subject, eventId, aggregateId, duration before/after |
| `LoggingConsumeMiddleware` | Log subject, eventId, processing duration, errors |
| `TracingPublishMiddleware` | Inject trace context (x-request-id, correlation-id) into NATS headers if supported |
| `TracingConsumeMiddleware` | Extract trace context from message, continue span |
| `IdempotencyConsumeMiddleware` | Check idempotency key (e.g. eventId) before invoking handler; skip if already processed |

### 10.4 Adapter integration

`NatsPublisher` and `NatsConsumer` accept optional middleware arrays:

```ts
constructor(
  private readonly nc: NatsConnection,
  private readonly options?: {
    publishMiddleware?: PublishMiddleware[]
    consumeMiddleware?: ConsumeMiddleware[]
  }
) {}
```

### 10.5 Idempotency middleware contract

Requires an injectable `IdempotencyPort` (or similar) from the domain. The event-bus package defines the middleware interface; the domain provides the implementation. The middleware calls `idempotency.isProcessed(eventId)` before invoking `next`; if true, returns early.

## 11. Phase V2.1 — Declarative NestJS Consumer Module

**Goal:** Reduce boilerplate across all services; no more manual `OnApplicationBootstrap` + `NatsConsumer` wiring.

### 11.1 Decorator

Create `src/nest/decorators/event-consumer.decorator.ts`:

```ts
export const EVENT_CONSUMER_METADATA = Symbol('EVENT_CONSUMER_METADATA')

export interface EventConsumerOptions {
  subject: string
  queue: string
  expectedVersion?: number
}

export function EventConsumer(
  subject: string,
  options: Pick<EventConsumerOptions, 'queue'> & { expectedVersion?: number }
): MethodDecorator {
  return (target, propertyKey) => {
    Reflect.defineMetadata(EVENT_CONSUMER_METADATA, {
      subject,
      queue: options.queue,
      expectedVersion: options.expectedVersion,
      handlerMethod: propertyKey
    }, target.constructor)
  }
}
```

### 11.2 Consumer registry

Create `src/nest/event-consumer.registry.ts`:

- Scans NestJS module for providers with `EVENT_CONSUMER_METADATA`
- Registers each as a subscription: `NatsConsumer.subscribe(subject, (envelope) => instance[handlerMethod](envelope))` with the given queue

### 11.3 Module

Create `src/nest/nats-consumer.module.ts`:

```ts
@Global()
@Module({
  imports: [],
  providers: [
    NatsConsumerRegistryService,  // OnModuleInit: discovers @EventConsumer, instantiates NatsConsumer, wires handlers
  ],
  exports: [NatsConsumerRegistryService]
})
export class NatsConsumerModule {}

// NatsConsumerModule.forRoot(options?: { queuePrefix?: string }): DynamicModule
```

### 11.4 Usage in domain service

```ts
@Injectable()
export class TrackUploadedConsumer {
  constructor(private readonly generateFingerprint: GenerateFingerprintUseCase) {}

  @EventConsumer(TrackEvent.Uploaded, { queue: 'petrified-workers' })
  async handle(envelope: EventPrimitive<TrackUploadedPayload>): Promise<void> {
    await this.generateFingerprint.execute({
      eventId: envelope.eventId,
      trackId: envelope.payload.trackId,
      // ...
    })
  }
}
```

No `OnApplicationBootstrap`, no manual `NatsConsumer` instantiation. The module discovers and wires consumers at bootstrap.

### 11.5 Dependency

Requires `@nestjs/common` (already a peer dependency). Optional: `reflect-metadata` for decorator metadata.
